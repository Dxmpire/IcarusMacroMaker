Quick Start Guide

From zero to your first working macro in a few minutes.

1 — Installation

ℹ️ Icarus Macro Maker requires Python 3.9 or newer on Windows 10/11. If you don't have Python, download it from python.org and make sure to check "Add Python to PATH" during installation.
1
Download the ZIP

Grab the latest release from the downloads page or directly from GitHub Releases. Extract it to a folder of your choice.

2
Install dependencies

Open a terminal inside the extracted folder and run:

pip install -r requirements.txt
3
Launch the app

Run the main entry point:

python main.py

The Icarus Macro Maker window will open along with the HUD overlay.

2 — Create your first macro

1
Open the Macros page

Click Macros in the sidebar. You'll see the macro list for your active profile (Default by default).

2
Create a new macro

Click + New Macro. Give it a name and choose a category. The editor opens automatically.

3
Add a condition

Click + Add Condition and choose how the macro triggers — for example, Action From Key. Then click Set Key on the condition to bind it to a key or button.

4
Add actions

Click + Add Action and pick an action type. Add as many as you need — key presses, delays, mouse moves, typed text. Drag rows to reorder them.

5
Save and test

Click Save. Make sure the macro is enabled (toggle in the macro list). Press your trigger key — the macro runs.

3 — Pixel Color Match (Reactive macros)

Reactive macros fire automatically when a pixel on screen matches a color — no key press needed. They live on the Reactions page in the sidebar.

1
Go to the Reactions page

Click Reactions in the sidebar. Click + New Macro.

2
Set the Pixel Color Match condition

The condition is added automatically. Click Edit on it to open the pixel editor. Click anywhere on screen to set the watch coordinate, or type exact X/Y values. Use the eyedropper to capture the target color.

3
Configure tolerance and radius

Tolerance controls how closely the color must match (0 = exact). Radius watches a circular area of nearby pixels instead of a single point, useful for anti-aliased edges.

4
Set the poll interval

In the Reactions page header, use the Check every dropdown to set how often the pixel is sampled (1–20 ms). Lower = more responsive, higher = lower CPU use.

4 — Configuring the HUD

The HUD is a floating overlay that stays on top of other windows. Customize it from the Settings page.

Position & Size

Choose a corner (Top Left, Top Right, Bottom Left, Bottom Right), the display to show it on, and custom width/height values.

Opacity & Border

Drag the opacity slider to make the HUD as subtle or bold as you need. Toggle the border on or off.

Click-Through

Enable click-through mode so mouse clicks pass through the HUD to whatever is behind it — essential for gaming overlays.

Panel Toggles

Toggle each HUD panel individually: title, profile name, clock, uptime, macro summary, macro list, run counts, pixel watch status, controller state.

Tips & common questions

My macro fires too slowly — how do I speed it up?

Replace Delay steps with shorter values or remove them entirely. For Type Phrase, switch to the Fastest preset or enable Instant mode.

Can I run multiple macros at the same time?

Yes. All enabled macros in the active profile are monitored simultaneously. Their conditions are evaluated independently.

How do I import a macro someone shared with me?

On the Macros page, click Import Macro and select the .json file. It is added to your current profile immediately.

The pixel watcher is using a lot of CPU.

Increase the poll interval on the Reactions page. A value of 5–10 ms is a good balance between responsiveness and CPU cost.

Where are my macros and settings saved?

Everything is stored in app/storage/ as JSON files. Back up that folder to preserve your profiles and macros.
